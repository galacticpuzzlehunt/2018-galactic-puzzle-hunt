Solution: Cookie Clicker
Answer: LEMON PIE

Written by Colin Lu

This puzzle is presented as a minimalist game, in which the player only has one means of input (clicking the cookie), and limited instructions or feedback.

For each level, the player must press the button at specific times in order to pass. Whenever the player first makes a mistake in their inputs, an “explosion” sound plays, resetting the level. This is intended to give players immediate feedback and enable to narrow down the correct rule for the inputs. The level titles and the sound effect upon clicking the cookie are intended to give additional hints as to what to do.

As the player begins each level, they will likely hear the “explosion” sound frequently, but will slowly narrow in on the rule. The rules are intended to be very simple and natural, with enough leniency in place so that the player is very likely to pass each level once they figure out what to do.

After the player finishes the warm-up levels and reaches level 1, they can use the arrow keys to jump between levels. This allows the player to “break in” to the encodings in various different ways.

The solutions for each level:

Warm-up 1SpeedyThe title suggests clicking quickly. To pass the level, click 60 times in under 10 seconds.
Warm-up 2QuickeningThe title suggests clicking increasingly quickly. To pass this level, the player must click with gaps of decreasing length.
Warm-up 3HaltingThe title of this puzzle suggests clicking at a decreasing rate. To pass this level, the player must click with gaps of increasing length.
Warm-up 4ClockThe title suggests timing similar to a clock. To pass the level, the player must hold for one second (actually 0.7-1.3 seconds, to give some leniency), and then leave a gap of one second (again actually 0.7-1.3 seconds), and repeat several times.
Level 1HorseThe title, as well as the sound effect the cookie makes upon being clicked, clue Morse code. Input HORSE in morse code to pass. The system recognizes presses of less than than 0.5 seconds as dots, and presses of more than 0.5 seconds as dashes. Gaps between dots/dashes in the same letter need to be less than 0.5 seconds, and gaps between letters need to be more than 0.5 seconds but less than 2.5 seconds.
Level 2MarathonThe title clues an analogue of “run-length encoding”. For this system, quickly pressing the button 13 times corresponds to “M”, pressing the button once corresponds to “A”, and so on. Gaps between presses for the same letter need to be less than 0.5 seconds, and gaps between letters need to be more than 0.5 seconds.
Level 3PasskeyThe title clues ASCII. The player must input Passkey in ASCII, which is 1010000,1100001,... Hold for more than 0.5 seconds to represent a “1”, and tap to represent a 0. The gaps between presses can be of any length.
Level 4FaucetThe title clues tap code. The player must input FAUCET in tap code. As before, consecutive taps must have gaps of less than 0.5 seconds, and gaps between consecutive taps need to be at least 0.5 seconds.

For the final “test” levels, the player is not told what they need to input, but in each case the required input is only two letters of the corresponding encoding, so it can be determined with the aid of the feedback system quickly. Each test corresponds to the “level” of the same number.

Test 1Input LE in morse code, using the same system as level 1.
Test 2Input MO using the scheme from level 2.
Test 3Input np using the ASCII encoding. Lowercase is used because the non-leading letters in the ASCII level were also lowercase.
Test 4Input IE using the tap code encoding.

By concatenating the answers to the “test” levels, we get the answer, LEMON PIE.

Author’s Notes

Incremental games such as the original Cookie Clicker have often been described (including by the creator of Cookie Clicker himself) as “non-games” due to their lack of substantial gameplay. With this in mind, I wanted to make a real and challenging puzzle game that only uses the cookie clicking as input for this Cookie Clicker themed hunt. I was also partially inspired by old electronic puzzles with relatively minimal instructions and input options, such as the Nemesis Factor.

It proved more difficult than expected to come up with reasonable puzzles within this constraint, as it is difficult to make a puzzle reasonably solvable but non-trivial in this environment. The first versions of the puzzle simply gave the answer at the end, but this was changed to something more like the current puzzle early on. While encodings such as morse code are frequently used in puzzle hunts, I thought it would be novel to have solvers input the encodings in a way perhaps more similar to their traditional use.

The version of the puzzle used in the hunt is in obfuscated javascript, to make it somewhat more difficult to solve the puzzle by reading the code. It is still possible to de-obfuscate the code and solve the puzzle without actually playing the game, but this isn't really easier than solving the puzzle “normally”, and it seems like very few teams did this.
